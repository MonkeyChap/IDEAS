<p>The measures model in IDEAS is deliberately kept to a minimum. It simply provides a framework for defining measure types, measures and a mechanism for representing a given measure. It also defines a few commonly used measure types such as mass, frequency, length, etc., but an IDEAS implementation can extend this set by simply creating new instances of MeasureType.<br/></p><p><br/></p><p>The measure model is quite simple. Measures are types - e.g. 2kg is the set of all things that weigh two kilograms. The members of these sets are always individuals. Measures have MeasureTypes - e.g. mass, length, etc. A given measure may have more than one representation. For example, the 2kg measure might be represented as "2.0kg", "2 kilos", "2.2046lbs", etc.<br/></p><p><br/></p><p>IDEAS does not make any attempt to reconcile units of measure against each other, it simply provides a framework for defining a property, giving it a value, and assigning the property to a thing. IDEAS also provides a few commonly used measures, and their corresponding SI unit.<br/></p><p><br/></p><p>The other aspect of measures not detailed by this pattern is the detail about precisely what the measure is of. For example, the length of a car can be measured bumper to bumper (fender to fender for our colonial chums), or by its wheelbase. The measure pattern described here is not concerned with precisely what is being classified, so it is up to the user to be precise about what they apply a measure to.  <br/></p><p><br/></p><p>The two main aspects of this measure pattern are shown in the example below.
